The body · the moat

Five shared organs. Everything else is distribution.

The body lives in the current oraclequant repo + Base44 + this VPS. Each organ already exists in some form — unification is extraction, not invention.

ORGAN 01

The Brain

OracleQuant's multi-model ensemble — IQR outlier removal, confidence-weighted fusion, Platt calibration — generalized per user. Every prediction is scored against real outcomes and recalibrated nightly.

ORGAN 02

Venice Gateway

The single LLM access layer (~/aiproxy) — one key, shared cache, shared rate limits. Every shirt calls the Brain through it. The key never enters the repo tree.

ORGAN 03

Contracts

The nervous system from workspace-contracts: UserRef, UsageEvent, Signal, WalletEvent, AgentEvent. One identity, one event bus every shirt reports into.

ORGAN 04

Risk Engine

Vendored from the trading bots: EV math, quarter-Kelly sizing, drawdown guard. Venue-blind, paper-first — any shirt that touches money uses the same spine.

ORGAN 05

The Shell

One design system, one app-switcher, one Brain widget. Extracted from the canonical Base44 template so a single token change re-skins all six shirts in one commit.

Merged correctly · reversible

One trunk, six branches — merged by contract, not copy-paste.

The classic mistake is letting apps become long-lived branches. Here, branches are only ever for migration and features. Each shirt joins in three reversible steps, and nothing user-visible breaks along the way.

STEP 1

Contract

The app keeps running where it is but adopts the shared contracts — UserRef login, UsageEvent/Signal emission into the Brain. Purely additive. Rollback = delete the adapter.

STEP 2

Layer

Pages move into the trunk as a route namespace (/markets, /life…) via git subtree — full history preserved — with its own entity namespace and one accent token, rendered in the shared shell.

STEP 3

Archive

Once contract parity is proven, the old repo is frozen with a pointer README. Duplicates fold into their canonical; spec-only repos archive after their decisions are captured.

The live-money gateHeadless services (Autopilot, aiproxy) never move into the shell — only their control surfaces do. Trading engines merge to trunk in paper mode only. Enabling real money requires a manual live/* promote plus a secret file outside the repo tree plus a runtime flag. No ordinary merge can flip real money on.
Execution order · safe sequence

The pullback: set the launch up perfectly, then let it fly.

A real sequence — each step ships value and is independently reversible. The working apps (Markets, Prism, OverlayFlow, the bots) never go dark.

  1. 1 · Trunk prep
    Extract the Brain service (ensemble + per-user calibration) out of oraclequant; merge oraclequant-copy and archive it; ship the shell around the existing Markets pages. Markets is shirt #1 by definition.
  2. 2 · Contracts on the pilots
    prism, emberscot, overlayflow adopt UserRef + UsageEvent/Signal emission. Additive adapters only — rollback is removing the adapter.
  3. 3 · Autopilot online
    Stand up crypto-trading-bot as a headless paper service, vendoring the kalshi engine. Add the /autopilot control surface. First cross-shirt proof: Markets predictions scored by Autopilot outcomes.
  4. 4 · Commerce joins
    prism pages move into /commerce; promote its billing stack up into the body as the shared payments layer for all shirts.
  5. 5 · Life consolidation
    Fold celestial-guidance + inkwell-prophet + cosmic-passport (+ auraguide) into one /life. Their prompts become Brain question types; the birth-profile becomes Brain context.
  6. 6 · Pocket alignment
    overlayflow-aiproxy merges into the shared Venice gateway; overlayflow-android becomes a thin renderer of the universal prediction card over the contracts.
  7. 7 · Forge + retirement sweep
    emberscot becomes /forge using deck's local-first bus/checkpoint patterns. Archive the long tail after salvaging any prompts/schemas into Brain question types. Every archive gets a pointer README.
